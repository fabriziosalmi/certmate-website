Security Best Practices

Authentication

  • Strong Bearer Tokens: Use cryptographically secure tokens (32+ characters)
  • Token Rotation: Regularly rotate API tokens
  • HTTPS Only: Always use HTTPS in production

File Permissions

  • Private keys: 600 permissions
  • Certificate directories: 700 permissions
  • Backup encryption recommended for sensitive data

Network Security

  • Implement firewall rules to restrict API access
  • Use reverse proxy (nginx/haproxy) for production
  • Enable audit logging for compliance
