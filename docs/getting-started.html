Getting Started

Get CertMate up and running in under 5 minutes

üöÄ Quick Start with Docker

Prerequisites

  • Docker 20.10+
  • Docker Compose 2.0+
  • Domain with DNS managed by a supported provider

1. Clone and Setup

# Clone the repository
git clone https://github.com/fabriziosalmi/certmate.git
cd certmate

# Copy environment template
cp .env.example .env

2. Configure Environment

Edit .env file with your credentials:

# üîí Required: API Security
API_BEARER_TOKEN=your_super_secure_api_token_here_change_this

# üåê DNS Provider Configuration (choose one)
# Option 1: Cloudflare (Recommended for beginners)
CLOUDFLARE_TOKEN=your_cloudflare_api_token_here

# Option 2: AWS Route53
# AWS_ACCESS_KEY_ID=your_aws_access_key
# AWS_SECRET_ACCESS_KEY=your_aws_secret_key

# Option 3: Azure DNS
# AZURE_SUBSCRIPTION_ID=your_azure_subscription_id
# AZURE_TENANT_ID=your_tenant_id
Docker Hub: Pre-built multi-platform images are available at fabriziosalmi/certmate

3. Deploy

# Start all services
docker-compose up -d

# Check status
docker-compose ps

# View logs
docker-compose logs -f certmate

4. Access CertMate

Service URL Description
Web Dashboard http://localhost:8000 Main interface
API Docs http://localhost:8000/docs/ Swagger UI
Health Check http://localhost:8000/health Service health

5. Create Your First Certificate

Using the Web Interface:

  1. Navigate to http://localhost:8000
  2. Go to Settings and configure your DNS provider
  3. Add your domain (e.g. example.com)
  4. Click "Create Certificate"

Using the API:

curl -X POST "http://localhost:8000/api/certificates/create" \
  -H "Authorization: Bearer your_api_token_here" \
  -H "Content-Type: application/json" \
  -d '{"domain": "example.com"}'

üì¶ Alternative Installation Methods

Python Virtual Environment

Ideal for development and testing:

# Create and activate virtual environment
python3 -m venv certmate-env
source certmate-env/bin/activate

# Install dependencies
git clone https://github.com/fabriziosalmi/certmate.git
cd certmate
pip install -r requirements.txt

# Run the application
python app.py

Kubernetes

For container orchestration:

kubectl apply -f kubernetes/deployment.yaml
kubectl apply -f kubernetes/service.yaml
kubectl apply -f kubernetes/ingress.yaml
Security Note: Always change the default API bearer token before deploying to production!
